# فهم شامل لتطبيق إدارة كروت الشبكات اللاسلكية

## نظرة عامة على التطبيق

### الغرض الأساسي
تطبيق ويب متكامل لإدارة مبيعات كروت الشبكات اللاسلكية (WiFi Cards) يساعد أصحاب المحلات على:
- تتبع المبيعات والمخزون
- إدارة الحسابات والمدفوعات
- متابعة المصروفات والأرباح
- إنشاء التقارير المالية

### التقنيات المستخدمة
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla)
- **UI Framework**: Bootstrap 5
- **التخزين**: LocalStorage, IndexedDB
- **المكتبات**: 
  - moment.js للتواريخ
  - jsPDF للتصدير PDF
  - xlsx للتصدير Excel
  - html2pdf للطباعة

## الوظائف الأساسية

### 1. إدارة المحلات
- **إضافة محل جديد**: اسم، رقم هاتف، نوع السعر
- **أنواع الأسعار**: تجزئة، جملة، موزعين
- **حساب الأرصدة**: تلقائي بناءً على المبيعات والمدفوعات
- **البحث والفلترة**: حسب الاسم، الرقم، نوع السعر

### 2. إدارة المبيعات
- **نوعان من المبيعات**:
  - بيع باقات (مع خصم من المخزون)
  - مبالغ مخصصة (بدون تأثير على المخزون)
- **تتبع الأسعار**: حسب نوع سعر المحل
- **التحقق من المخزون**: تنبيه عند انخفاض الكمية

### 3. إدارة المصروفات
- **أنواع متعددة**: قابلة للتخصيص
- **البحث والترتيب**: حسب التاريخ، النوع، المبلغ
- **التقسيم لصفحات**: 10 عناصر لكل صفحة
- **العمليات الجماعية**: حذف متعدد

### 4. نظام التقارير
- **تقرير الأرباح**: المبيعات - المصروفات
- **تقرير الديون**: أرصدة المحلات
- **تقرير الشركاء**: توزيع الأرباح
- **تقارير مفصلة**: لكل محل على حدة
- **التصدير**: Excel, PDF, طباعة

### 5. النسخ الاحتياطي والمزامنة
- **النسخ المحلي**: تصدير/استيراد JSON
- **مزامنة GitHub**: عبر Gist API
- **التشفير**: للبيانات الحساسة
- **العمل Offline**: Service Worker

## تدفق البيانات

### هيكل البيانات الرئيسي
```javascript
data = {
  stores: [],      // قائمة المحلات
  packages: [],    // أنواع الباقات
  inventory: [],   // المخزون
  sales: [],       // المبيعات
  payments: [],    // المدفوعات
  expenses: [],    // المصروفات
  expenseTypes: [] // أنواع المصروفات
}
```

### دورة حياة البيانات
1. **الإدخال**: عبر النماذج المختلفة
2. **التحقق**: validation أساسي
3. **التخزين**: LocalStorage + IndexedDB
4. **المعالجة**: حسابات وتقارير
5. **العرض**: جداول وإحصائيات
6. **التصدير**: مختلف الصيغ

## التحديات الحالية

### 1. الأمان
- تشفير ضعيف (XOR)
- مفتاح ثابت في الكود
- لا توجد حماية CSRF

### 2. الأداء
- حسابات متكررة
- لا يوجد تخزين مؤقت
- ملفات كبيرة جداً

### 3. الموثوقية
- معالجة أخطاء ضعيفة
- لا يوجد نظام معاملات
- مشاكل في المخزون

## الخلاصة

التطبيق يمثل حلاً متكاملاً لإدارة مبيعات كروت الشبكات مع إمكانيات جيدة للتطوير. رغم وجود بعض التحديات التقنية، إلا أن الأساس قوي ويمكن تحسينه ليصبح منتجاً احترافياً.

✅ يعمل بدون اتصال
