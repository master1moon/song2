# إصلاح مشكلة التاريخ في التطبيق

## وصف المشكلة
عند إدخال تاريخ مخصص (مثل 20/07/2025) في أي قسم من أقسام التطبيق، كان التاريخ يتم حفظه دائماً كتاريخ اليوم بدلاً من التاريخ المدخل.

## الأقسام المتأثرة
- المصروفات
- الباقات والأسعار
- كمية الكروت (المخزون)
- البقالات والمحلات
- المبيعات
- التسديدات

## أسباب المشكلة

### 1. مشكلة في دالة formatDateEn
كانت الدالة تقطع أول 10 أحرف فقط من التاريخ باستخدام `.slice(0, 10)` مما كان يسبب مشاكل في معالجة التواريخ بصيغ مختلفة.

### 2. مستمعي أحداث غير ضرورية
كان هناك مستمعي أحداث (`change` و `blur`) على حقول التاريخ تقوم بتغيير القيمة المدخلة.

### 3. معالجة خاطئة للتاريخ الافتراضي
كان يتم استخدام `|| today` مباشرة مع `formatDateEn` مما يؤدي لتمرير قيمة خاطئة للدالة.

## الحلول المطبقة

### 1. تحسين دالة formatDateEn
```javascript
function formatDateEn(dateStr) {
  if (!dateStr) return '';
  
  // تحويل الأرقام العربية إلى إنجليزية
  const englishDate = toEnglishDigits(dateStr);
  
  try {
    if (typeof moment !== 'undefined') {
      // جرب صيغ مختلفة للتاريخ
      const formats = [
        'YYYY-MM-DD',
        'YYYY-M-D',
        'DD/MM/YYYY',
        'D/M/YYYY',
        'DD-MM-YYYY',
        'D-M-YYYY',
        moment.ISO_8601
      ];
      
      const m = moment(englishDate, formats, true);
      if (m.isValid()) {
        return m.format('YYYY-MM-DD');
      }
    }
  } catch (_) {}
  
  // معالجة يدوية للصيغ المختلفة
  // ... باقي الكود
}
```

### 2. إزالة مستمعي الأحداث غير الضرورية
تم إزالة مستمعي `change` و `blur` من حقول التاريخ لأن حقول HTML5 date تتعامل مع الصيغة تلقائياً.

### 3. تصحيح معالجة التاريخ الافتراضي
تم تغيير:
```javascript
const date = formatDateEn(document.getElementById('dateField').value || today);
```

إلى:
```javascript
const date = document.getElementById('dateField').value ? formatDateEn(document.getElementById('dateField').value) : today;
```

## النتيجة
الآن يمكن إدخال التواريخ بالصيغ التالية وسيتم حفظها بشكل صحيح:
- 20/07/2025
- 20-07-2025
- 2025-07-20
- 2/7/2025
- 2025-7-2

## اختبار الحل
1. اذهب إلى قسم المصروفات
2. أضف مصروف جديد
3. أدخل تاريخ مخصص (مثل 20/07/2025)
4. احفظ المصروف
5. ستجد أن التاريخ تم حفظه بشكل صحيح كـ 2025-07-20

نفس الطريقة تعمل في جميع الأقسام الأخرى.
