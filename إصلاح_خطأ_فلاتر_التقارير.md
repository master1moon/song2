# إصلاح خطأ فلاتر التقارير

## وصف المشكلة
في قسم التقارير، كانت قائمة المحلات في الفلاتر لا تعمل بشكل صحيح ولا تظهر المحلات المضافة.

## سبب المشكلة
كان هناك تضارب في أسماء العناصر:
- دالة `updateReportStores()` كانت تبحث عن عنصر بـ id `reportStoreSelect`
- لكن العنصر الفعلي في HTML له id `reportsStoreFilter`
- نتيجة لذلك، كانت الدالة تفشل في العثور على العنصر ولا تضيف المحلات

## الحل المطبق

### 1. تصحيح دالة updateReportStores في reports.js
```javascript
function updateReportStores() {
  // البحث عن العنصر الصحيح reportsStoreFilter بدلاً من reportStoreSelect
  const select = document.getElementById('reportsStoreFilter');
  if (!select) return;
  
  // حفظ القيمة الحالية
  const currentValue = select.value;
  
  // مسح الخيارات الحالية باستثناء "جميع المحلات"
  while (select.options.length > 1) select.remove(1);
  
  // إضافة المحلات
  data.stores.forEach(store => {
    const option = document.createElement('option');
    option.value = store.id; 
    option.textContent = store.name; 
    select.appendChild(option);
  });
  
  // استعادة القيمة المحددة إذا كانت لا تزال موجودة
  if (currentValue && [...select.options].some(opt => opt.value === currentValue)) {
    select.value = currentValue;
  }
}
```

### 2. تصحيح نفس الدالة في index.html

## التحسينات الإضافية
1. **حفظ القيمة المحددة**: الآن يتم حفظ المحل المحدد عند تحديث القائمة
2. **التحقق من وجود العنصر**: إضافة فحص للتأكد من وجود العنصر قبل محاولة التعديل عليه

## النتيجة
الآن عند إضافة محلات جديدة أو تحديث البيانات، ستظهر جميع المحلات في قائمة الفلاتر في قسم التقارير بشكل صحيح.

## اختبار الحل
1. اذهب إلى قسم "البقالات والمحلات"
2. أضف محل جديد
3. اذهب إلى قسم "التقارير"
4. افتح قائمة "المحل" في الفلاتر
5. ستجد المحل الجديد موجوداً في القائمة
