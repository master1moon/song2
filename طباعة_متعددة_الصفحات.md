# حل مشكلة الطباعة متعددة الصفحات

## 🎯 التحسينات المطبقة:

### 1. **فصل الإجمالي النهائي:**
- تم نقل الإجمالي من `<tfoot>` إلى `<div>` منفصل
- يظهر فقط مرة واحدة في نهاية التقرير
- محمي من التكسر بـ `page-break-inside: avoid`

### 2. **حماية العناصر من التكسر:**
```css
tr { page-break-inside: avoid; }
.date-group-header { page-break-after: avoid; }
```

### 3. **ترقيم الصفحات (نظرياً):**
```css
@page {
    @bottom-center {
        content: "صفحة " counter(page) " من " counter(pages);
    }
}
```

## ⚠️ القيود الحالية:

### مشكلة الإجماليات الجزئية:
لا يمكن تطبيق "إجمالي جزئي لكل صفحة" بشكل مثالي باستخدام CSS/HTML فقط لأن:
1. المتصفحات لا تدعم JavaScript أثناء الطباعة
2. لا يمكن معرفة مكان فواصل الصفحات مسبقاً
3. CSS لا يمكنه حساب المجاميع ديناميكياً

## 💡 الحلول البديلة:

### 1. **الحل الحالي (مبسط):**
- الإجمالي النهائي يظهر مرة واحدة في النهاية
- كل صف يحتوي على الرصيد المتحرك
- يمكن تتبع الرصيد من أي صفحة

### 2. **حل متقدم (يتطلب مكتبة):**
استخدام مكتبة مثل:
- **jsPDF** مع **html2canvas**
- **Puppeteer** (server-side)
- **pdfmake**

### 3. **حل يدوي:**
تحديد عدد الصفوف لكل صفحة (مثلاً 30 صف) وإضافة إجماليات يدوياً

## 📝 التوصيات:

### للاستخدام الحالي:
1. **الرصيد المتحرك كافٍ** - كل صف يظهر الرصيد
2. **رأس التاريخ واضح** - سهل معرفة بداية كل يوم
3. **الإجمالي النهائي** - في آخر صفحة فقط

### لتطوير مستقبلي:
إذا كانت الإجماليات الجزئية ضرورية:
```javascript
// استخدام jsPDF
const doc = new jsPDF();
// حساب الصفوف لكل صفحة
// إضافة إجماليات جزئية
// حفظ كـ PDF حقيقي
```

## ✅ النتيجة الحالية:
- التقرير يُطبع بشكل صحيح
- لا تكرار للإجمالي النهائي
- العناصر محمية من التكسر
- الرصيد واضح في كل صف
